# Product Vision: x-growth-copilot

> **Живой документ.** Сюда складываются и уточняются мысли по продукту; на него ориентируются при принятии решений и при разработке.

---

## 🎯 Главная цель

Достижение 10K+ качественных фолловеров в X (Twitter) до конца 2026 года.
Качество важнее количества. Мы строим долгосрочный актив, который будет работать на репутацию. Проект должен быть **надёжным и масштабируемым** — не развалиться завтра.

---

## 🧠 Основная концепция (Thinking Partner)

Это приложение — не бездумный генератор спам-реплаев. Это **AI-копилот для глубокого мышления**.

**Главная задача продукта:** помогать в **поиске разных углов для атаки** тем, которые хочется осветить в X. Будь то чужие посты (QT, комментарии) или **свой оригинальный пост** на актуальную/трендовую тему.

**Суть процесса:** углубление в тему и **поиск того, о чём в посте открыто не сказали** — но при включении критического мышления кое-что можно заметить. ИИ-агенты помогают найти это "кое-что". После диалога (долгого или короткого) собираем полноценный пост.

- **Реплаи/комменты:** сначала раскрыть суть чужого поста, найти нужный угол, чтобы комментарий **вызвал реакцию автора или читателей** — и только потом упаковать в текст.
- **Оригиналы/треды/статьи:** те же принципы (глубокий разбор, углы атаки, диалог), но с **своими правилами** под каждый тип контента (посты продвигает X, люди читают; разные типы постов в разные дни; треды и статьи — отдельные правила).

**Ключевой Workflow (единый чат, без отдельного агента-упаковщика):**

1. **Инициализация:** При каждом запуске чат уже загружен: Voice Bank, правила для выбранного типа контента (Reply по умолчанию), anti-patterns. При переключении типа контента — пересобирает контекст.
2. **Глубокий анализ (Phase 1 — Intelligence):** Первое сообщение запускает автоматический разбор: суть, скрытый смысл, уязвимости, рекомендуемые углы атаки / MODE (название).
3. **Диалог и штурм (Phase 2 — Mining):** Пинг-понг с ИИ. По ходу диалога пользователь выделяет удачные фразы и добавляет их в **Заметки (Notes)**. Чат **видит заметки в реальном времени** как дополнительный контекст и использует их для улучшения следующих вариантов (эквивалент "вот эта фраза мне нравится, давай запомним и будем ее использовать в финальном варианте, двигаемся дальше").
4. **Финализация:** Когда финальный вариант готов, пользователь выделяет текст → появляется **popup** → **"Add to queue"**. Текст одновременно: (а) копируется в буфер обмена, (б) отправляется в ближайший свободный слот планировщика. Пользователь идёт в X и вставляет текст. **Отдельного агента-упаковщика нет** — весь процесс происходит внутри одного чата.

## 💻 Архитектура UI

Вместо 5 раздельных страниц — **один мощный рабочий экран** с тремя колонками. Это сохраняет фокус на потоке мышления, меняются только системные промпты (в зависимости от выбранного типа контента в селекторе перед отправкой).

- **Левый сайдбар (Работа с историей и планом):**
  - **Drafts** — незавершённые диалоги. Каждый диалог — это незаконченный черновик, который ведет к публикации.
  - **Scheduled** — слоты стратегии (например, "2 поста в день, 4 сессии реплаев"). Визуальное давление пустых ячеек.
  - **Settings** — настройки Voice Bank, API ключи и конфигурация стратегии.
- **Центр (Главный рабочий экран):**
  - Чат с моделью (спарринг-партнером). Оригинальный пост/URL пользователь вставляет в первое сообщение — отдельного блока «карточка поста» сверху нет.
  - Внизу: поле ввода + селектор `[Тип контента ▾]` (Reply / Post / Thread / Article) + кнопка Отправить. _Выбор типа контента в самом начале меняет фокус анализа агента._
- **Заметки / Notes:**
  - **Привязан к текущему диалогу.** При переключении на другой Draft — заметки переключаются тоже. При возврате к старому Draft — заметки восстанавливаются.
  - Зона, куда собираются **цитаты из диалога**. Единственный способ добавить заметку — выделить текст в чате → **floating popup** → "Add to Notes" (context menu по выделению не делаем).
  - Чат **видит эти заметки в реальном времени** и использует их как приоритетный контекст для генерации следующих вариантов используя их максимально дословно.
  - Очищается при начале нового диалога (или вручную).

## 🤖 Логика Агентов (Разные процессы мышления)

Единый чат без отдельного агента-упаковщика и без промежуточного переключения промптов. Один промпт ведёт весь диалог от анализа до финального варианта. Voice Bank, anti-patterns и Notes предзагружены при инициализации и обновляются в реальном времени.

**Архитектура промптов: 2 файла-функции + константы.**
Граница между промптами проходит по _направлению работы_ (реакция на чужое vs создание своего), а не по типу контента:

- **`analyst-reply.ts`** → для Reply (весь диалог: Phase 1 анализ + Phase 2 спарринг)
- **`analyst-post.ts`** → для Post / Thread / Article (весь диалог + условный блок правил формата по типу)
- **Константы** (`modes.ts`, `anti-patterns.ts`, `hooks.ts`) импортируются в оба промпта.

**Reply (Реагируешь на чужое):**

- Агент анализирует: какой это тип поста (хайп, боль, провокация)?
- Рекомендует **MODE** (A - Skeptic, B - Solidarity, C - Nuance, D - Insider, E - Universal).
- Учитывает **алгоритмическую цель**: размер/вес аккаунта автора (крупный лидер мнений → profile clicks от его аудитории; равный аккаунт → author engagement) и контекст (лента / Community).
- Цель: короткий текст, который вызовет ответ автора.

**Post / Thread / Article (Создаешь своё):**

- Агент анализирует: где второе дно темы? Что все игнорируют?
- Рекомендует: Hook (из Top-7 типов), углы атаки.
- Условный блок правил формата: Post (длина, hook scoring), Thread (структура, cliffhangers), Article (длинная форма, разделы).
- Цель: остановить скролл, репосты. Проверка по Checklist (10/10) и генерация VISUAL блока.

## 📥 Ввод контента

- **URL:** вставка ссылки на твит → парсинг текста через бесплатный шлюз.
- **Текст вручную:** копирование поста и вставка в поле — обязательно. Нужно и когда парсинг не сработал, и когда хочешь отвечать на конкретный фрагмент, а не на весь твит.

---

## 📐 Типы контента (единый нейминг: Reply / Post / Thread / Article)

У каждого типа — свои правила (чтобы X продвигал, а люди читали):

- **Reply** — реплай/комментарий под чужим постом. Сначала суть и угол для реакции автора/аудитории.
- **Post** — оригинальный пост. Свои правила, разные типы в разные дни.
- **Thread** — тред (серия связанных твитов). Отдельные правила структуры.
- **Article** — длинная статья. Свои правила.

---

## ⚙️ Ограничения и Правила Игры (Алгоритм X 2026)

- **Author Engaged Replies (Вес 75.0):** Цель — спровоцировать реакцию автора оригинального поста. Один ответ автора стоит алгоритмически в 5.5 раз больше обычного реплая.
- **Burst-penalty (Защита от спама):** Не вываливать 30 реплаев за час. Жесткое разнесение сессий (3-5 качественных ответов за присест, перерыв 2-3 часа).
- **Качество контента:** ИИ-след должен быть абсолютно незаметен. Строгие ограничения по длине, паттернам и тону. Никаких шаблонных фраз вроде "100%", "Exactly", "Great point".
